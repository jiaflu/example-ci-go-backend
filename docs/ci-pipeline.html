Go Backend CI Pipeline

Automated quality gate for tests, coverage, binary build, and container image validation.

CI quality gate
1

Trigger

Pull request or push to main starts the workflow.

2

Go setup

Checkout, Go 1.22, and module cache.

3

Format

gofmt check blocks unformatted code.

4

Static checks

go vet catches common correctness issues.

5

Tests

go test ./... validates all packages.

6

Coverage gate

Coverage reports generated and checked at 80% minimum.

7

Race check

go test -race checks concurrent safety.

8

Binary build

Builds the Go backend executable.

9

Buildx

QEMU and Buildx prepare multi-arch builds.

10

Docker image

Builds linux/amd64 and linux/arm64 container image.

11

Registry placeholder

Push step is present but disabled until registry exists.

12

Artifacts

Reports, binary, and OCI image tar are uploaded.

CI Outputs

coverage-summary.md Client-readable coverage result.
coverage.html Line-by-line coverage visualization.
example-ci-go-backend Linux backend binary.
image.tar Multi-architecture OCI image artifact.
GitHub Actions validates code quality before merge and container readiness before deployment.
